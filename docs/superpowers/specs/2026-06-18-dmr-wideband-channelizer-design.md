# DMR 宽带信道化扫描（全频段覆盖）设计

**日期：** 2026-06-18
**范围：** 在现有 `realtime/` 实时层之上新增**宽带信道化前端**（第一级），把硬件一次采集的 60–70MHz 宽带 IQ 流切成 N 个等宽、重叠的 ~2.5MHz 子带；活跃子带交给现有 2.5MHz 实时管线（第二级）解码。本期用仿真验证离线正确性，硬件接入仅替换数据源。
**不在范围内：** SDR 硬件实测（`SoapyWidebandSource` 占位）；实时 60Msps 吞吐达标（本期只保证离线正确性，性能调优留待硬件阶段）；语音解码（沿用 `CallRecord.voice_raw` 占位接口）；**跳频/retune**（硬件瞬时带宽 ≥ 目标频段，不再需要）。

---

## 1. Context（为什么做这个）

### 1.1 现状与硬件前提

现有 `RealtimeScanner` 锁定单一固定带宽（瞬时带宽 = 复采样率）。新硬件一次可采集 **60–70MHz** 瞬时带宽——只要关心的 DMR 频段落在这一段内，接收机**一次就同时收下所有信号**，无需调谐扫描。

这把问题从"调度/跳频"彻底换成了**计算吞吐**：60Msps（每秒约 6–7 千万复样点）的数据如何处理得过来。先前的跳频方案（`SweepController`、盲区状态机、时间-频率耦合）随之**全部作废**。

### 1.2 目标

在 60–70MHz 宽带上**系统性覆盖**全部潜在 DMR 信道，自动发现活跃通话、解析信令（LC Header / CSBK / Terminator / Late Entry）、输出带**绝对射频频率**的结构化通话记录。先用仿真验证整条链路，硬件接入仅替换 `WidebandIQSource` 实现。

### 1.3 核心设计原则：穷举栅格，不反推信号

DMR 是窄带协议，信道严格落在 **12.5kHz 固定栅格**上（代码中 `channel_grid_hz=12500` 即此）。70MHz / 12.5kHz ≈ 5600 个潜在信道槽位，**位置完全确定**；哪个槽位有信号是随机且时变的，但槽位本身已知。

因此信道化前端必须**系统性、无差别覆盖整个 60–70MHz 的栅格**（与任何具体信号无关），而非"看真实录制里出现了什么就处理什么"。覆盖能力由栅格几何决定，不被测试样本的偶然性限制。

**对验证的含义**：覆盖性是确定性几何问题，合成数据能把每个槽位、每个边界条件（尤其骑在子带分界上的信道）精确测到，比真实录制的随机覆盖更彻底、更可控。本期仅能合成数据，这反而正好。

### 1.4 为什么不直接切 5600 个 12.5kHz 窄带（单级方案被否）

单级信道化器 bin 宽 12.5kHz，刀口必须正好压在每个栅格上。一旦接收机中心频率有偏（本振误差、或 fc 非 12.5kHz 整数倍），每个信道都骑在 bin 边界、被劈成两半全盘崩溃。单级把"对齐"这个最难的事压在唯一一级、无补救层，脆弱。

**采用两级（粗筛 + 精调，类超外差）**：第一级粗信道化（宽子带，粗到不在乎信道是否对齐刀口）+ 第二级现有管线（Welch 找峰 → DDC 到真实峰值 → `frontend` 峰值自校准吸收剩余频偏）。中心频偏与 straddling 在两级间被自然化解。

## 2. 全局约束

- 解调核心 `core/dsp.py`、`core/decoder.py`、`core/burst_type.py` **不修改**。
- `scanner._decode_loop`、`realtime/worker.decode_window`、`multiprocessing.Pool` 派发**复用**，不重写。
- 信道化器只依赖 **numpy + scipy**（环境现有 numpy 2.2.6 / scipy 1.15.3），不引入 GNU Radio 等重依赖；与 `core/dsp.py` 全 scipy 风格一致。
- 子带数 N、过抽样因子、原型滤波器参数均为运行时可配；本期取经验值，硬件阶段实测调优。
- **过抽样必需**：相邻子带必须重叠，保证骑在子带分界上的 12.5kHz 信道至少在一个子带内完整（化解 straddling）。
- 全速回归测试用 `throttle=False`；数据文件缺失用 `pytest.skip()`，不得用裸 `return`。
- **性能边界（诚实声明）**：本期目标为离线正确性；实时 60Msps 吞吐达标需硬件阶段实测调优（N、过抽样因子、worker 数），不在本期承诺。

## 3. 架构与组件边界

```
60–70MHz 宽带 IQ 流
   │
   ▼ 【第一级·新增】PolyphaseChannelizer（唯一新重活）
   │   一次 FFT 切成 N 个等宽、重叠的 ~2.5MHz 子带
   │   过抽样 → 子带边缘重叠，骑边界信道不被劈裂
   │
   ▼ N 个 ~2.5MHz 子带 → 能量检测挑出活跃子带（少数几个）
   │
   ▼ 【第二级·复用】现有 realtime 管线，解码逻辑一行不改
   │   活跃子带 → Detector(找峰即解) → worker.decode_window → SessionAggregator
   │
   ▼ 通话记录（绝对射频频率）
```

**核心不变量：解调核心（`core/`、`scanner._decode_loop`、`worker.decode_window`）一行不改；新增的全是"宽带 → 子带"的信道化前端。**

### 文件清单

```
realtime/channelizer.py        create  PolyphaseChannelizer：宽带块 → N 个重叠子带
                                       (scipy.firwin 原型滤波器 + 多相 reshape + FFT + 过抽样相位修正)
realtime/wideband_source.py    create  WidebandIQSource 抽象 + FileWidebandSource(仿真) + SoapyWidebandSource(占位)
realtime/scanner_rt.py         modify  主循环前端插入信道化:宽带流 → N 子带 → 各喂现有 detector/worker/aggregator
realtime/aggregator.py         modify  归并键改为绝对射频频率(子带中心 + 检测偏移)
utils/synthesis.py             modify  合成更宽(如 10–20MHz)的栅格场景文件用于验证
tests/test_channelizer.py      create  子带中心 + 过抽样重叠 + 骑边界信道完整 + 栅格覆盖 + 流式衔接 + bin→频率映射
tests/test_wideband_e2e.py     create  端到端:宽带文件 → 信道化 → 解出多个分散在不同子带的通话
```

### 各单元职责

- **PolyphaseChannelizer**：纯 DSP 组件。输入一块宽带 IQ + 配置（N、过抽样因子、原型滤波器），输出 N 个子带 IQ 及各子带绝对中心频率。`core/dsp.py` 风格，全 scipy/numpy，每一步可独立单测。持有多相分支延迟线状态以支持流式分块。
- **WidebandIQSource**：宽带数据源抽象（语义升级为"60MHz 级宽带"）。仿真版读宽带文件；硬件版发 SDR 命令。**无 tune**——一次罩住全部，下游对仿真/硬件无感知。
- **RealtimeScanner（改造）**：主循环前端加"宽带块 → 信道化 → N 子带"层，每个活跃子带走现有 detector→worker→aggregator。**第二级解码逻辑一行不改。**
- **SessionAggregator（小改）**：归并键改为绝对射频频率（子带中心 + 子带内检测偏移），使分散在不同子带的通话各自独立、同一通话不裂开。

## 4. PFB 信道化器内部数据流

### 4.1 一次性初始化（构造时）

```
原型滤波器：scipy.signal.firwin(N*M, cutoff≈1/N)   # 长度 N*M 低通,定义子带形状(过渡带/旁瓣=子带隔离度)
按 n mod N 拆成 N 个多相分支,reshape 成 (N, M)       # N=子带数(取2的幂,如32); M=每分支抽头数(如12)
```

### 4.2 每块流式处理（状态延续）

```
宽带块 → 换向器:样点轮流发到 N 个分支(过抽样则步进 N/2)
       → 每分支短 FIR(沿用上一块遗留延迟线,块间状态延续)
       → N 点 FFT(每轮一次)
       → 过抽样相位修正(每次 FFT 输出乘随时间变化的相位因子)
       → fftshift + bin→绝对频率映射
       → 输出 N 个子带,各已抽取到 ~fs/N(临界) 或 ~2fs/N(过抽样)
```

数学等价于"用那条长原型滤波器对每个子带做完整滤波 + 下变频 + 抽取"，但运算量约为**一次 N 点 FFT + N 个短 FIR**，而非 N 次全速率 DDC。FFT 提供效率，多相分支提供选择性，抽取由 FFT 结构免费内建。

### 4.3 过抽样化解 straddling（关键）

```
临界抽样(步进N):   [子带0][子带1][子带2]    边界信道被劈裂
过抽样2×(步进N/2): [子带0  ]
                     [  子带1  ]            相邻重叠50%,任何信道至少在一个子带完整
                       [  子带2  ]
```

换向步进 N → N/2 即得 2× 过抽样。代价：输出数据量翻倍、需相位修正。对"不漏骑边界信道"是必须的。

### 4.4 四个实现风险点（登记）

1. **原型滤波器设计**——`firwin` 一行，但截止/长度决定子带隔离度。
2. **bin → 绝对频率映射**——含 fftshift、过抽样相位，最易写错；单测以已知单音校验。
3. **过抽样相位修正**——WOLA 最绕一环；子带中心相对换向起点有偏移，每次 FFT 输出须乘时变相位因子，否则子带频谱旋转。
4. **流式状态衔接**——分支 FIR 延迟线块间延续，否则块边界毛刺；单测以"分两块 vs 整块输出一致"校验。

### 4.5 参数关系与可配性

**核心绑定关系：`子带宽度 = 输入采样率 / N`。** 三者绑定，不能各自任取。"2.5MHz 子带"不是目标而是结果（60Msps / 24 ≈ 2.5MHz）——选 N 让子带宽度落在现有第二级管线能吃的范围，是为**复用**，不是物理限制。

```
N=12 → 5.00 MHz/子带   N=24 → 2.50 MHz   N=32 → 1.88 MHz   N=48 → 1.25 MHz   （@60Msps）
```

**N（= FFT 点数 = 子带数）的选取：**
- **偏好 2 的幂**：FFT 在 N=16/32/64 最快；N=24 等非 2 的幂 scipy 也能算（混合基），但略慢。实务默认 **N=32**（2 的幂 + 子带宽度适中）。
- **不能太小**：子带过宽 → 一个子带塞入过多 12.5kHz 信道、单子带处理变重，失去分治意义。
- **不能太大**：子带过窄（接近 12.5kHz）→ 退化为单级直切窄带，撞回中心频偏 straddling；且过抽样后子带数翻倍，开销上升。
- **经验区间 16–64，默认 32**；具体值硬件阶段按真实采样率与信道密度实测定。

**输入采样率变化时的自适应：** 信道化器从数据源读 `sample_rate`（同现有 `IQSource.sample_rate` 做法），子带宽度 = 采样率/N 自动跟变；第二级 `resample_poly` 按新子带率动态推导抽取因子，自动适应。信道化器本身**不改**。仅当要**锁定**子带宽度恒定时才需改 N（N=采样率/目标子带宽），N 改则 FFT 重新规划一次（初始化时算，不影响流式）。

**边界（超出本期前提）：** 若目标 DMR 频段 > 硬件瞬时带宽（如频段散布 100MHz、硬件仅 60MHz），则"一次罩住全部"前提不成立，需在 `WidebandIQSource` 上加一层粗调谐（整体搬移 60MHz 窗口）；信道化器与第二级仍复用，但本期不做（§1.1 前提为目标频段 ≤ 硬件瞬时带宽）。

### 4.6 与第二级衔接

每个子带采样率 ~2fs/N。例：60Msps / 32 × 2(过抽样) ≈ 3.75Msps/子带，略高于现有 2.5MHz 管线输入，由 `decode_window` 现成的 `resample_poly` 吸收（或调 N 使子带率更贴近 2.5MHz）。子带绝对中心频率随块输出，供 aggregator 构造绝对频率坐标。

## 5. 错误处理与监控

| 场景 | 处理 |
|------|------|
| 文件/流耗尽 | `read_chunk()` 返回 None → 主循环排空 → 聚合器 flush 未关闭 session（timeout）→ 退出 |
| worker 解码异常 | 复用 `decode_window` 内 try/except，单窗失败返回 []，不拖垮池 |
| 子带无能量 | 能量检测跳过，不派发解码（省算力） |
| 信道骑子带边界 | §4.3 过抽样重叠保证至少一个子带完整 |
| 中心频偏/发射机频漂 | 第二级 Welch 找峰即解 + `frontend` 峰值自校准吸收（复用） |
| 频率微抖 | detector 按 `channel_grid_hz` 量化；aggregator 按 `fo_bucket` 归并（复用） |
| 块边界毛刺 | §4.4 多相延迟线块间状态延续 |

## 6. 验证策略

沿用 `throttle=False` 全速回归 + 数据缺失 `pytest.skip()`。核心命题：**信道化器系统覆盖全栅格、且不漏骑边界的信道**——确定性几何问题，合成数据精确测每个边界条件。

**仿真数据：** 扩展 `synthesize_scenario` 造更宽（先 10–20MHz，不必满 60MHz）的栅格场景文件，把测试信号精确摆在子带正中、相邻子带、以及**正好骑两子带分界**的槽位。测试用短时长压体积。

**单元层（channelizer，重点）：**
- **子带中心正确**：子带中心放单音 → 该 bin 出现峰、其余安静。
- **过抽样重叠**：两子带分界处放单音 → **相邻两子带都看到完整信号**（straddling 核心断言）。
- **栅格覆盖**：扫覆盖全带宽的一组频点 → 每个都落进至少一个子带、能量守恒（无覆盖空洞）。
- **流式衔接**：同一信号分两块 vs 整块送入 → 子带输出在块边界一致（无毛刺）。
- **bin→绝对频率映射**：已知频率单音 → 输出标注绝对频率与真值一致。

**集成层：**
- 信道化 → 现有 detector/worker：一子带内放一路 DMR → 解出正确 SRC/DST。
- 绝对频率归并：分散在不同子带的两路 → aggregator 输出两条独立 CallRecord、频率正确。

**端到端：**
- 合成 10–20MHz 含多路 DMR（分散不同子带、含一路骑边界）→ 整条链路 → 断言**全部**被发现、各带正确绝对射频频率。**铁证：信号跨度超过单个 2.5MHz 子带，唯有信道化能全部抓到。**

**验证脚本：** 扩展 `scanner_rt.py` CLI 加 `--wideband --nsub N --oversample 2`，命令行直接跑宽带文件看输出。

## 7. 关键决策记录

1. **不跳频**：硬件瞬时带宽（60–70MHz）≥ 目标 DMR 频段，一次罩住全部；先前 sweep 方案作废。
2. **两级架构**：第一级 PFB 粗信道化（宽子带 + 过抽样重叠，不要求对齐栅格）+ 第二级现有 2.5MHz 管线（找峰即解、自校准吸收频偏）。类超外差，化解中心频偏与 straddling。
3. **穷举栅格、不反推信号**：信道化按固定 12.5kHz 栅格系统覆盖全带宽，与具体信号无关；合成数据验证覆盖性。
4. **PFB 用 scipy 积木自拼**：`firwin`/`fft` 由 scipy 封装，仅多相 reshape + 过抽样相位修正自写（约几十行）；不引入 GNU Radio 重依赖。
5. **过抽样必需**：换向步进 N/2 得 2× 过抽样、相邻子带重叠，保证骑边界信道完整。
6. **本期离线正确性优先**：实时 60Msps 吞吐留待硬件阶段实测调优。
