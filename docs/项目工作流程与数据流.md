# 项目工作流程与数据流

本文描述当前工程在重构后的整体工作流程、数据流动路径和主要中间产物。重点覆盖离线多制式扫描主链路，同时说明 DMR、P25、dPMR 三个协议模块内部如何消费前端数据，以及 realtime/宽带部分如何复用离线核心。

## 1. 入口总览

当前工程有 `1 + 3` 个离线入口：

| 入口 | 命令 | 作用 |
|------|------|------|
| 统一主入口 | `python scanner.py <iq_file>` | 对输入 IQ 同时运行 DMR、P25、dPMR 检测与解码 |
| DMR 单独入口 | `python -m dmr <iq_file>` | 只运行 DMR 离线解码链路 |
| P25 单独入口 | `python -m p25 <iq_file>` | 通过主 scanner 只启用 P25 |
| dPMR 单独入口 | `python -m dpmr <iq_file>` | 通过主 scanner 只启用 dPMR |

主入口支持协议过滤：

```bash
python scanner.py data/dmr_1_78125.rawiq --protocol dmr
python scanner.py data/p25_1_78125.rawiq --protocol p25
python scanner.py data/dpmr_1_48000.rawiq --protocol dpmr
```

## 2. 核心模块职责

| 模块 | 职责 |
|------|------|
| `common/io.py` | 读取 interleaved int16 IQ，识别文件名中的采样率 |
| `common/dsp.py` | 协议无关插值、窄带 FSK 前端 |
| `scanner.py` | 离线主编排：读文件、候选搜索、DDC、重采样、前端生成、协议分发、去重、输出 |
| `protocols.py` | 协议 registry、协议名归一化、统一分发、协议相关去重 key |
| `dmr/` | DMR 常量、DSP、突发恢复、LC/CSBK/Late Entry 解码、DMR 单入口 |
| `p25/` | P25 frame sync、NID、HDU/LDU/LC、session 组装 |
| `dpmr/` | dPMR FS1/FS2 同步、符号恢复、CCH、Color Code、session 组装 |
| `core/` | 兼容层，旧 import 转发到 `dmr.*` |
| `realtime/` | 实时/宽带信道化链路，当前通过 `scanner._decode_loop()` 兼容复用离线协议分发 |

## 3. 离线主链路数据流

统一入口 `scanner.scan_file()` 是离线主链路。整体数据流如下：

```text
rawiq/bvsp-derived IQ file
    │
    ▼
common.io.read_rawiq()
    │  np.ndarray[complex]，归一化到约 [-1, 1]
    ▼
detect_sample_rate()
    │  从文件名识别 fs，例如 *_78125.rawiq -> 78125
    ▼
窄带 / 宽带分支
    │
    ├─ 窄带 fs <= 200 kHz
    │      │
    │      ▼
    │   resample_poly -> 48 kHz
    │
    └─ 宽带 fs > 200 kHz 或未知
           │
           ▼
        Welch PSD 粗扫 -> 候选频偏 fos
           │
           ▼
        对每个 fo: DDC -> resample_poly -> 48 kHz
    │
    ▼
协议前端
    │
    ├─ dmr.dsp.frontend()      -> y       # DMR/P25 共用 48 kHz FSK 鉴频输出
    └─ dpmr.dsp.frontend_dpmr() -> y_dpmr # dPMR 窄带前端
    │
    ▼
protocols.decode_all(y, frontends={"dPMR": y_dpmr})
    │
    ├─ DMR decoder
    ├─ P25 decoder
    └─ dPMR decoder
    │
    ▼
PDU list
    │
    ├─ dpmr.decoder.filter_stable_pdus()
    └─ protocols.deduplicate_pdus()
    │
    ▼
打印结果 / 可选 JSON
```

### 3.1 `scanner.py` 函数级工作流程

`scanner.py` 是离线主入口和编排层。它不直接实现协议解码算法，而是负责把输入 IQ 转换成协议模块可消费的前端数据，再把协议输出统一过滤、去重和格式化。

整体调用关系：

```text
main()
  └─ scan_file()
       ├─ _normalize_protocol_names()
       ├─ read_rawiq()
       ├─ detect_sample_rate()
       ├─ _psd_blind_search()          # 宽带/未知采样率路径
       ├─ _process_candidate()         # 宽带候选路径
       │    ├─ _resample_factors()
       │    ├─ dmr.dsp.frontend()
       │    ├─ dpmr.dsp.frontend_dpmr()
       │    └─ _decode_protocol_frontends()
       │         └─ protocols.decode_all()
       ├─ _process_narrowband()        # 窄带路径
       │    ├─ _resample_factors()
       │    ├─ dmr.dsp.frontend()
       │    ├─ dpmr.dsp.frontend_dpmr()
       │    └─ _decode_protocol_frontends()
       ├─ filter_stable_pdus()
       ├─ protocols.deduplicate_pdus()
       ├─ _print_results()
       └─ _write_json()                # 可选
```

#### 3.1.1 入口与采样率

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `main(argv=None)` | `argv: list[str] | None` | `int` | CLI 入口。解析目标文件、`--protocol`、`--fo`、`--json`，逐文件调用 `scan_file()`。 |
| `scan_file(path, freq_list=None, output_json=None, protocol_names=None)` | `path: str`；可选频偏列表、JSON 输出路径、协议名列表 | `list[dict]` PDU | 离线扫描主函数。负责读 IQ、判断窄带/宽带路径、调用协议、去重、打印和 JSON。 |
| `detect_sample_rate(path)` | `path: str` | `int | None` | 从文件名中解析采样率，例如 `data/dmr_1_78125.rawiq -> 78125`。解析失败返回 `None`。 |
| `_normalize_protocol_names(protocol_names)` | `list/tuple/set[str] | None` | `set[str]` | 调用 `protocols.normalize_protocol_names()`，把 `dmr/p25/dpmr` 转成规范名 `DMR/P25/dPMR`。 |

`scan_file()` 的输入分支：

```text
freq_list is not None
  -> 使用用户指定频偏，逐个调用 _process_candidate()

fs is None or fs > 200_000
  -> 视为宽带或未知采样率文件
  -> _psd_blind_search() 找候选频偏
  -> 每个候选调用 _process_candidate()

fs <= 200_000
  -> 视为窄带文件
  -> _process_narrowband()
```

#### 3.1.2 宽带候选搜索

| 函数 | 输入 | 输出 | 中间产物 | 说明 |
|------|------|------|----------|------|
| `_psd_blind_search(iq, fs)` | `iq: np.ndarray` complex；`fs: float` | `list[float]` 候选频偏 Hz | `f`、`psd`、`psd_db`、`nf`、`peaks` | 对宽带 IQ 做 Welch PSD，使用 `median(psd_db) + 15 dB` 检测峰值。 |

处理细节：

```text
iq
  -> scipy.signal.welch(..., nperseg=4096, return_onesided=False)
  -> fftshift(f), fftshift(psd)
  -> psd_db = 10 * log10(psd + 1e-12)
  -> nf = median(psd_db)
  -> find_peaks(height=nf + 15, distance=20)
  -> [f[p] for p in peaks]
```

输出的 `fo` 是相对于输入 IQ 中心频率的候选频偏。后续每个 `fo` 都会单独 DDC 到零频附近。

#### 3.1.3 重采样与前端生成

| 函数 | 输入 | 输出 | 中间产物 | 说明 |
|------|------|------|----------|------|
| `_resample_factors(source_sample_rate, target=Fs_dec)` | `source_sample_rate: float`；`target: float` 默认 48 kHz | `(up, down)` | `gcd(up, down)` | 计算 `scipy.signal.resample_poly()` 使用的最简整数重采样比例。 |
| `_process_candidate(iq, fo, fs_in, protocol_names=None)` | 原始 IQ、候选频偏、输入采样率、启用协议集合 | `list[dict]` PDU | `t`、`iq_shifted`、`iq_dec`、`y`、`y_dpmr` | 宽带候选处理：DDC、重采样、生成协议前端、调用协议解码，并给 PDU 加 `_fo_hz`。 |
| `_process_narrowband(iq, fs_in=None, protocol_names=None)` | 原始窄带 IQ、输入采样率、启用协议集合 | `list[dict]` PDU | `iq_dec`、`y`、`y_dpmr` | 窄带处理：必要时重采样到 48 kHz，然后生成协议前端并解码。 |

`_process_candidate()` 数据流：

```text
iq, fo, fs_in
  -> t = arange(len(iq)) / fs_in
  -> iq_shifted = iq * exp(-j * 2π * fo * t)
  -> iq_dec:
       if fs_in ~= 48 kHz: 直接使用 iq_shifted
       elif fs_in ~= 2.5 MHz: resample_poly(iq_shifted, UP_FACTOR, DOWN_FACTOR)
       else: _resample_factors(fs_in) -> resample_poly(iq_shifted, up, down)
  -> y = dmr.dsp.frontend(iq_dec, fs=48 kHz)
  -> y_dpmr = dpmr.dsp.frontend_dpmr(iq_dec, fs=48 kHz)
  -> _decode_protocol_frontends(y, y_dpmr, protocol_names)
  -> 每个 PDU 附加 pdu["_fo_hz"] = fo
```

`_process_narrowband()` 与 `_process_candidate()` 的区别是：窄带样本默认已经在目标信道附近，不做 DDC，也不会给 PDU 附加 `_fo_hz`。

#### 3.1.4 协议分发兼容层

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `_decode_protocol_frontends(y, y_dpmr=None, protocol_names=None)` | DMR/P25 前端 `y`；dPMR 前端 `y_dpmr`；协议集合 | `list[dict]` PDU | 构造 `frontends={"dPMR": y_dpmr}`，调用 `protocols.decode_all()`。 |
| `_decode_loop(y)` | `y: np.ndarray` float | `list[dict]` PDU | 兼容 realtime 旧路径，直接调用 `protocols.decode_all(y)`。 |
| `_decode_dmr_loop(y)` | `y: np.ndarray` float | `list[dict]` DMR PDU | 兼容旧 DMR 私有入口，内部转调 `dmr.engine._decode_dmr_loop()`。 |

协议分发实际发生在 `protocols.py`：

```text
protocols.decode_all(y, protocol_names=names, frontends={"dPMR": y_dpmr})
  -> registry 顺序遍历 DMR / P25 / dPMR
  -> DMR 和 P25 默认消费 y
  -> dPMR 消费 frontends["dPMR"]
```

#### 3.1.5 后处理、去重和输出

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `filter_stable_pdus(pdus)` | `list[dict]` | `list[dict]` | dPMR 稳定性过滤。该函数来自 `dpmr.decoder`，会保留稳定 color code 和高/中质量 dPMR PDU；非 dPMR PDU 透传。 |
| `protocols.deduplicate_pdus(pdus)` | `list[dict]` | `list[dict]` | 使用协议相关 key 去重。DMR 按 SRC/DST/type/频偏桶，P25 按 NAC/type/frame bucket，dPMR 按 SRC/DST/color_code/frame bucket。 |
| `_print_results(pdus)` | `list[dict]` | `None` | 根据 `pdu["protocol"]` 分发到 P25/dPMR/DMR 格式化输出。 |
| `_write_json(pdus, path)` | PDU 列表、输出路径 | `None` | 写 JSON 文件。会剔除 `raw_bits` 字段。 |

`scan_file()` 的最终输出始终是去重后的 `unique: list[dict]`，即使同时指定了 `output_json`，返回值也不会变。

#### 3.1.6 打印格式化函数

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `_format_dpmr_result(pdu, fo_str="")` | dPMR PDU、可选频偏字符串 | `str` | 格式化 dPMR 类型、SRC/DST、color code、FS 类型、极性和 CCH 摘要。 |
| `_format_dpmr_cch(cch_records)` | `list[dict | None]` | `str` | 把 dPMR CCH record 列表格式化为 `FN/IDH/M/V/F/E/RES/SLD` 字段。 |
| `_format_p25_result(pdu, fo_str="")` | P25 PDU、可选频偏字符串 | `str` | 根据 P25 PDU type 格式化 HDU/LDU1/LDU2/CALL/NID。 |
| `_p25_detail(pdu)` | P25 PDU | `str` | 生成 P25 细节字段，包括 NAC/DUID/BCH/LC/HDU/ES 等。 |

DMR 当前没有单独 `_format_dmr_result()`，在 `_print_results()` 的默认分支中直接格式化：

```text
[LC_HEADER] PROTO=DMR SRC=... DST=... FLCO=... FID=...
```

#### 3.1.7 `scanner.py` 中间产物汇总

| 阶段 | 变量 | 类型 | 生命周期 | 是否输出到文件 |
|------|------|------|----------|----------------|
| 读文件 | `iq` | complex ndarray | `scan_file()` 内 | 否 |
| 采样率判断 | `fs` | `int | None` | `scan_file()` 内 | 否 |
| 宽带粗扫 | `fos` | `list[float]` | 宽带分支内 | 否 |
| DDC | `iq_shifted` | complex ndarray | `_process_candidate()` 内 | 否 |
| 重采样 | `iq_dec` | complex ndarray | `_process_candidate()` / `_process_narrowband()` 内 | 否 |
| DMR/P25 前端 | `y` | float ndarray | 单次候选/窄带处理内 | 否 |
| dPMR 前端 | `y_dpmr` | float ndarray | 单次候选/窄带处理内 | 否 |
| 协议输出 | `all_pdus` | `list[dict]` | `scan_file()` 内 | 可经 JSON 输出 |
| 去重输出 | `unique` | `list[dict]` | `scan_file()` 返回 | 可经 JSON 输出 |
| 频偏标记 | `_fo_hz` | `float` | 宽带候选 PDU 内 | JSON 保留 |
| 原始比特 | `raw_bits` | `bytes` | PDU 内 | JSON 输出时删除 |

## 4. 主要数据对象

| 名称 | 类型 | 产生位置 | 含义 |
|------|------|----------|------|
| `iq` | `np.ndarray` complex | `common.io.read_rawiq()` | 原始复数 IQ，来自 int16 I/Q 交错文件 |
| `fs` | `int | None` | `scanner.detect_sample_rate()` | 输入采样率，通常从文件名推断 |
| `fos` | `list[float]` | `scanner._psd_blind_search()` | 宽带粗扫得到的候选频偏 |
| `iq_shifted` | `np.ndarray` complex | `scanner._process_candidate()` | 已按候选频偏 DDC 到零中频附近的 IQ |
| `iq_dec` | `np.ndarray` complex | `resample_poly` | 重采样到 48 kHz 的窄带 IQ |
| `y` | `np.ndarray` float | `dmr.dsp.frontend()` | FM 鉴频、去 DC、归一化后的符号尺度样点流 |
| `y_dpmr` | `np.ndarray` float | `dpmr.dsp.frontend_dpmr()` | dPMR 专用窄带前端输出 |
| `PDU` | `dict` | 各协议 decoder | 统一输出事件，包含协议、类型、源/目的、附加字段和原始 bits |
| `CallRecord` | dataclass | `realtime/aggregator.py` | realtime 聚合后的呼叫记录 |

统一 PDU 当前仍是 dict，约定字段如下：

```python
{
    "protocol": "DMR" | "P25" | "dPMR",
    "type": "...",
    "src": ...,
    "dst": ...,
    "ts": 0,
    "flco": "...",
    "fid": "...",
    "extra": {...},
    "raw_bits": b"...",
}
```

JSON 输出会去掉 `raw_bits`，避免二进制字段污染结果文件。

## 5. DMR 模块内部流程

DMR 的离线实现位于 `dmr/offline.py`、`dmr/dsp.py`、`dmr/decoder.py`。

```text
y
 │
 ▼
dmr.dsp.find_sync_positions()
 │  输出 [(center_sample, polarity, sync_type), ...]
 │
 ├─ DATA sync: DATA_MS / DATA_BS
 │      │
 │      ▼
 │   dmr.dsp.recover_burst()
 │      │  132 symbols，已按同步码仿射校准
 │      ▼
 │   dmr.decoder.decode_burst()
 │      │
 │      ├─ Slot Type Golay(20,8,7)
 │      ├─ BPTC(196,96)
 │      ├─ RS(12,9,4)
 │      └─ FullLinkControl / CSBK
 │
 └─ VOICE sync: MS_VOICE / BS_VOICE
        │
        ▼
     _lock_voice_phase()
        │
        ▼
     _recover_stepped_burst()，按 60 ms stride 恢复 A-F burst
        │
        ▼
     LateEntryCollector.feed()
        │
        ├─ Embedded Signalling
        ├─ LCSS fragment state machine
        ├─ VBPTC(128,72)
        └─ FiveBitChecksum
```

DMR 主要中间产物：

| 中间产物 | 含义 |
|----------|------|
| `positions` | DMR 同步位置、极性和同步类型 |
| `symbols` | 一个 132-symbol burst 的四电平符号 |
| `ba264` | 一个 DMR burst 对应的 264-bit bitarray |
| `slot_bits` | Slot Type 20 bit，用于 Golay 和 data type 判断 |
| `info196` | BPTC 输入 196 bit |
| `decoded[0:96]` | BPTC 输出，FLC/CSBK 的协议负载 |
| `LateEntryCollector._frags` | 语音突发中的 4 个 32-bit embedded LC fragment |

## 6. P25 模块内部流程

P25 的主入口是 `p25.decoder.decode(y)`。

```text
y
 │
 ▼
p25.sync.find_frame_sync()
 │  输出 P25SyncCandidate(fs_start, polarity, ncc)
 │
 ▼
p25.dsp.recover_symbols_from_fs()
 │  恢复 FS + NID 符号
 │
 ▼
slice_symbols_to_bits() + extract_nid_bits()
 │
 ▼
p25.nid.decode_nid()
 │  NAC / DUID / BCH 状态
 │
 ▼
p25.framing.frame_info_from_nid()
 │
 ├─ HDU: recover_full_frame() -> decode_hdu_hcw()
 ├─ LDU1: recover_full_frame() -> decode_ldu1_lc()
 └─ LDU2: recover_full_frame() -> decode_ldu2_es()
 │
 ▼
P25SessionAssembler.feed()
 │
 ▼
P25_NID / P25_HDU / P25_LDU1 / P25_LDU2 / P25_CALL PDU
```

P25 主要中间产物：

| 中间产物 | 含义 |
|----------|------|
| `P25SyncCandidate` | 帧同步起点、极性和 NCC 分数 |
| `nid_bits` | P25 NID BCH 输入位 |
| `P25NID` | NAC、DUID、BCH 校验/纠错状态 |
| `P25FrameInfo` | DUID 解释后的帧类型、是否语音/控制/终止 |
| `LinkControl` | LDU1 中恢复出的 SRC、DST/TGID、MFID、LCO |
| `HeaderCodeWord` | HDU 中恢复出的 MI、ALGID、KID、TGID |
| `EncryptionSync` | LDU2 中恢复出的加密同步字段 |
| `P25SessionAssembler` 状态 | 跨 HDU/LDU/TDU 组装一次通话 |

## 7. dPMR 模块内部流程

dPMR 的主入口是 `dpmr.decoder.decode(y_dpmr)`。dPMR 使用自己的窄带前端和符号恢复参数。

```text
y_dpmr
 │
 ├─ Header path
 │    │
 │    ▼
 │  find_fs1_sync()
 │    │
 │    ▼
 │  recover_frame_symbol_candidates()
 │    │
 │    ├─ decode_cch()
 │    └─ get_color_code()
 │
 └─ Voice path
      │
      ▼
    find_fs2_sync()
      │
      ▼
    recover_voice_fs2_symbol_candidates()
      │
      ▼
    split_voice_fs2()
      │
      ├─ CCH0
      ├─ Color Code
      └─ CCH1
      │
      ▼
    DPMRSessionAssembler.feed()
```

dPMR 主要中间产物：

| 中间产物 | 含义 |
|----------|------|
| `DPMRSyncCandidate` | FS1/FS2 同步起点、极性、NCC、同步类型 |
| `DPMRSymbolCandidate` | 符号序列、残差、SPS、相位、判决质量 |
| `CCHRecord` | CCH 解码结果，包括帧号、ID half、CRC/Hamming 状态 |
| `color_code` | dPMR color code |
| `quality` | CRC/Hamming/帧号组合得到的置信度摘要 |
| `DPMRSessionAssembler` 状态 | 根据 CCH 片段组装 SRC/DST |

`scanner.scan_file()` 在协议解码后调用 `filter_stable_pdus()`，根据稳定 color code 和质量字段过滤 dPMR 假阳性。

## 8. 去重和输出

协议去重逻辑在 `protocols.py`：

| 协议 | 去重 key |
|------|----------|
| DMR | `(protocol, src, dst, type, fo_bucket)` |
| P25 | `(P25, nac, type, frame_bucket)` |
| dPMR | `(dPMR, src, dst, color_code, frame_bucket)` |

其中：

- `fo_bucket = round(_fo_hz / 5000) * 5000`
- P25 `frame_bucket = round(fs_start / 8640)`
- dPMR `frame_bucket = round(fs_start / 3840)`

输出路径：

```text
PDU list
 │
 ├─ scanner._print_results() -> stdout
 │
 └─ scanner._write_json() -> JSON 文件
        │
        └─ raw_bits 字段被剔除
```

## 9. realtime 与宽带信道化数据流

本次模块化重构没有重排 `realtime/`，但实时路径仍能通过兼容函数复用离线核心。

窄带/宽带实时数据流：

```text
File/SDR IQ source
    │
    ▼
realtime.iq_source / wideband_source
    │
    ▼
RingBuffer
    │
    ▼
固定长度重叠 window
    │
    ▼
Detector / Channelizer
    │  输出活跃候选信道和频偏
    ▼
realtime.worker.decode_window()
    │
    ├─ DDC
    ├─ resample_poly -> 48 kHz
    ├─ dmr.dsp/core.dsp frontend 兼容入口
    └─ scanner._decode_loop()
            │
            └─ protocols.decode_all()
    │
    ▼
PDU list，附加 _fo_hz / _window_id
    │
    ▼
realtime.aggregator.SessionAggregator
    │
    ▼
CallRecord / on_call 回调
```

宽带信道化路径 `realtime/wideband_scanner.py` 会先把宽带 IQ 分成多个子带，再对子带窗口复用同样的 `decode_window()` 逻辑。

## 10. 文件级中间产物

| 路径 | 类型 | 说明 |
|------|------|------|
| `data/*.rawiq` | 输入样本 | int16 I/Q 交错原始样本 |
| `DMR_signal/*.bvsp` | 输入样本 | USRP/BVSP 宽带采集，需按 BVSP 头解析 |
| `output/*.png` | 调试图 | 历史调试脚本生成的 PSD、同步、星座/质量图 |
| `output/*.rawiq` | 合成/场景数据 | 宽带 demo 或合成器生成的中间场景 |
| `output/*.gif` | 可视化结果 | 宽带 demo 动画 |

多数协议中间产物只存在于内存中；除非运行 debug/visualize 脚本，否则不会自动写入 `output/`。

## 11. 推荐调试顺序

当某个样本解不出来时，建议按以下顺序定位：

1. 确认输入采样率：文件名是否包含 `_<fs>.rawiq`，否则 scanner 会按宽带默认路径处理。
2. 窄带样本先跑单协议入口，减少其他协议假阳性干扰。
3. 宽带样本先看 Welch 候选频偏，必要时用 `--fo` 手动指定候选。
4. DMR 优先检查同步位置、Golay、BPTC/RS。
5. P25 优先检查 Frame Sync、NID BCH、NAC 稳定性。
6. dPMR 优先检查 FS1/FS2、CCH CRC/Hamming、color code 稳定性。
7. realtime 问题先复现到离线 `scanner.py`，再检查 window 长度、step、候选频偏和 aggregator。
