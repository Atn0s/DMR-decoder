# DMR 信号解调工作报告

> Reed-Solomon(12,9,4) 校验。**并进一步实现了 Embedded Signalling 中途加入 (late entry)

> Late Entry」「第 11 节 正确性验证」。

## 1. 任务背景

- 在 2.4 MHz 带宽信号中随机放置了两路 DMR 和一路 P25 对讲机录音，目标是把其中的 DMR 信号解调出来。
- 目标文件：`synthesized_wideband_2.5MHz.rawiq`（2.5 MHz 采样，int16 IQ 交织）。
- 参考素材：`dmr_1_78125.rawiq` / `dmr_2_78125.rawiq`（原始单路 DMR，78125 Hz）、`p25_1_78125.rawiq`。
- 主处理脚本：`dmr_pipeline_v2.py`。
- 旁证：同一信号经 SDR++ FM 解调 → 虚拟声卡 → dsd-fme 已能正常出声，证明信号合法可解。

## 2. 处理流程（dmr_pipeline_v2.py）

| 阶段 | 内容 | 状态 |
|------|------|------|
| Stage 1 | 宽带 Welch 扫描找能量峰 → 候选频偏 | 正常 |
| Stage 2 | DDC + resample_poly(12/625) 降到 48 kHz + 线性相位 FIR | 正常 |
| Stage 3 | FM 鉴频 → 门控去 DC → 按 DEV_NOMINAL 缩放 → integrate&dump | 部分有效 |
| Stage 4 | BS/MS 同步波形归一化互相关 (NCC) | 正常 |
| Stage 5 | 周期性校验（语音 360ms / 数据 30-60ms）区分 DMR vs P25 | 正常 |
| Stage 6 | Gardner 定时 → 找数据同步码 → LC Header 解码 (Golay/BPTC/CRC-24) | **失败** |

## 3. 已确认的结论（实验证据）

1. **检测链 Stage 1-5 正确**：两路 DMR 均被定位为 MS/BS Sourced，P25 被正确拒绝。
2. **信号是 TDMA 突发结构**：参考文件 median amp≈0 不是噪声，而是时隙间空闲；
   突发间隔约 60 ms、每段约 28 ms，符合 DMR 双时隙 TDMA。必须先门控到单突发再解调。
3. **不是 FM click，也不是信噪比问题**：突发内 clickfrac=0，幅度正常，
   瞬时频率 std≈1268 Hz，与理想 4FSK 理论值（约 1449 Hz）吻合。
4. **同步定位正确**：数据同步码归一化相关有明确强峰，粗对齐没问题。
5. **旁证**：同一信号经 SDR++ FM 解调 → 虚拟声卡 → dsd-fme 已能正常出声，
   证明信号合法、协议层无误，问题完全在自研处理链内。

## 4. 待解决的核心问题

**符号判决层不收敛 → Golay(20,8,7) 全败 → 到不了 BPTC 和 CRC-24。**

- 把已知全为 ±3 的 24 个同步符号取出、做仿射校准后，符号位（sign）能中 20-22/24，
  但幅度散布在 ±0.5 ~ ±4，四电平（±1/±3）没有分开，RMSE≈0.5-0.6。
- 四电平直方图峰在 -1250 / -850 / +50 / +1450 Hz，不对称、内外比不是 1:3。
- 收窄积分窗 / 单点采样更差；cutoff、SPS、相位多档扫描，Golay 通过率始终 0-1。
- 误差沿突发位置完全平坦 → 排除采样率/定时漂移。

结论：逐样点鉴频输出在符号中心没有干净的四电平平台。

## 5. 根因分析（按可能性排序）

1. **判决器是固定门限 + 一次性仿射标定，不是自适应跟踪**（最致命）。
   现用死的 ±2/0 门限，每突发只拟合一次增益/偏置；真实信号 center/gain 在突发内会漂，
   内层 ±1 一偏就翻位。
2. **解码路径没有真正收敛的符号时钟环**：Stage 6 实际用"同步中心−66符号"的开环栅格，
   采样点落在过渡沿上，把内层符号拉向 0（直方图 0 处的伪峰来源）。
3. **信道滤波器过窄（6500 Hz）**：实测 DMR 占用带宽约 7.6 kHz，6.5 kHz 削掉外层边带，
   压缩外层、引入 ISI，应放宽到 9-10 kHz。
4. **载波/直流用 Welch 谱峰估计，对 4FSK 有偏**：残余直流平移四电平，叠加固定门限导致误判。
5. 匹配滤波形式存疑（次要）：boxcar 未必匹配发射脉冲成形，但非主因。

## 6. 2014 年录制 / 制式相关性

**制式不是失败原因。** DMR 空口二层（ETSI TS 102 361）2005-2007 定型，2014 已成熟；
同步码、BPTC(196,96)、Golay(20,8,7)、Voice LC Header 的 CRC-24 掩码 0x969696 均未变过，
dsd-fme 用同一套标准参数已解出。唯一与年代相关的是采样率 78125 Hz（=2.5e6/32，疑似当年
声卡采集），对 4800 baud 是非整数 16.276 SPS，使参考文件难处理；但目标文件抽取到 48 kHz
后是整数 10.0 SPS，反而干净——提示应直接在 48 kHz / 10-SPS 目标链上重做。

## 7. 解决方案（借鉴 dsd-fme，保留 Stage 1-5）

只重写"鉴频输出 → 符号"这一段：

1. 信道滤波放宽到 9-10 kHz，停止削外层符号。
2. center/直流改用突发内鉴频中位数，或命中同步码后用同步码重标定，不用谱峰。
3. 符号定时：10 SPS 整数栅格上跑真正收敛的环（Gardner / Mueller-Müller），
   或 dsd 式"中心几样点求和 + 小步时钟微调"，锁定符号中心。
4. **自适应四电平判决器**：维护滑动 max/min/center/umid/lmid，每命中同步码用其（全±3）
   刷新 max/min，门限随之更新，替换固定 ±2/0。
5. 定帧：符号域 sign 相关找同步码，容错匹配。
6. 验证指标用 Golay 通过率：先在单突发把它从 0 拉到接近 100%，过了再谈 BPTC / CRC-24。
7. 兜底：把各阶段 48 kHz 鉴频音频导出 WAV，与喂给 dsd-fme 的那路逐段对比，
   定位是"鉴频音频不同"还是"音频相同但符号环差"。

## 8. 下一步

从第 1+2 步（放宽滤波 + 同步码重标定 center）入手，在单突发上验证 Golay 通过率，
再逐步加入自适应判决器与符号时钟环。

```
候选 3 (+600.6 kHz): REJECTED (P25/噪声)
```

### 关键修改

1. **新增独立 LC 前端 `lc_front_end()`**：信道滤波 6.5→9.5 kHz（停止削外层 ±1944 Hz 符号），
   center 用 active 区鉴频中位数（替代有偏的 Welch 谱峰）。
2. **样点域定帧 `find_data_sync_positions()` + `recover_burst_symbols()`**：
   归一化互相关在样点域定位数据同步码，按同步中心 ±整数 SPS 栅格取 132 符号，
   亚符号相位用同步码 24 符号（已知 ±3）做仿射拟合扫描，选 4 电平残差最小相位。
   替代了原来不收敛的符号域 Gardner 开环路径。
3. **自适应四电平判决器 `adaptive_slice_bits()`**：每突发用 max/min/center/umid/lmid
   动态门限，替代固定 ±2/0 门限。
4. **根因纠正——24 位不是 CRC-24，而是 Reed-Solomon(12,9,4)**：
   ETSI TS 102 361-1 B.3.6，掩码 0x969696（B.3.12）。直接调用
   `okdmr ReedSolomon1294.check()` 验证，并用 `FullLinkControl.from_bits()` 解析字段。
   这解释了为何 `crc24_selftest.py` 永远搜不到匹配参数——前提（CRC-24）就是错的。

### 注意事项

- `crc24_selftest.py` 已作废（前提错误，应忽略或删除）。
- 解出的 SRC=1/DST=1 是合成信号用的测试 ID；4 帧独立超帧（间隔 60ms）一致且 RS 校验通过，
  确认解码真实可信。
- 78125 Hz 参考文件因非整数 16.276 SPS 较难处理；目标文件抽取到 48 kHz 后整数 10 SPS，

## 10. Embedded Signalling 中途加入 (Late Entry) — 已完成

实现脚本 `late_entry.py`（独立文件，import 复用 `dmr_pipeline_v2` 的引擎函数）。

### 应用场景

LC Header 只在通话**开始**出现一次。接收机若在通话**中途**介入，错过头帧，只能靠每个
语音突发中间嵌入的 Embedded Signalling (EMB) 碎片，跨 4 个突发重建同样的 LC 内容。

### 协议结构（ETSI TS 102 361-1）

- 语音超帧 = 6 个突发 A–F，周期 360 ms（每突发 60 ms = 2880 样点 @48k）。
- 突发 A 携带语音同步码；突发 B–E 中心 48-bit = 16-bit EMB 头 + 32-bit 信令碎片。
- EMB 头含 LCSS 状态，受 QR(16,7,6) 保护（仅护 16-bit 头）。
- 4 个连续碎片（First→Cont→Cont→Last）拼成 128 bit，经 VBPTC(128,72) 纠错
  → 72-bit LC + 5-bit CS5 校验和。

### 实现流程

1. **语音同步锚定** `find_voice_sync_anchor()`：NCC>0.7 定位 Burst A（每个=一超帧起点），
   实测 360 ms 周期，max NCC≈0.89。
2. **相位锁定** `lock_phase_from_anchor()`：用 Burst A 语音同步码拟合最优亚符号相位，
   传递给同超帧的 B–E（语音突发中心是嵌入信令、无已知图案可仿射，故沿用 A 的相位，
   增益由自适应判决器自校准）。
3. **EMB/LCSS 状态机** `decode_one_superframe()`：收集 First→Cont→Cont→Last 共 4×32 bit。
4. **VBPTC(128,72) 纠错** + `FullLinkControl.from_bits()` 解析。
5. **CS5 五位校验和门控**（关键）：CS5（B.3.11）保护全部 72-bit 重组 LC，比仅护 16-bit
   头的 EMB QR 强得多。CS5 不过即丢弃，滤掉所有拼接错帧。CS5 合法范围 0–30，收到 31
   直接判失败。

### 运行结果

| 输入 | 命中超帧 | 唯一 LC（CS5 通过后） |
|------|---------|----------------------|
| 干净 dmr_2 (53.8s) | 56 | SRC=1 DST=1 GroupVoiceChannelUser |
| 干净 dmr_1 (27.6s) | 52 | SRC=1 DST=1 GroupVoiceChannelUser |
| 合成-候选1 (dmr_1, -299.7kHz) | 23 | SRC=1 DST=1 |
| 合成-候选2 (dmr_2, +150.1kHz) | 7 | SRC=1 DST=1 |

合成文件能解出的超帧数远少于干净文件（23/7 对比 56/52），因为 synthesis.py 叠加了 20dB
AWGN，EMB 的 QR(16,7,6) 纠错弱，碎片易错；但 CS5 保证了**只要解出就一定正确**。

### 配置开关

- `EMB_TOLERANT=True`：QR 头校验失败仍收碎片（QR 只护 16-bit 头，容错收集 + CS5 终判更稳）。
- `CS5_STRICT=True`：要求重组 LC 的 CS5 通过（推荐，滤除全部拼接错帧）。

## 11. 正确性如何确认（多重独立校验）

解析结果的可信度不靠"看着像"，而是靠**多层独立的协议级校验 + 两条独立路径交叉验证**：

### 单帧级 FEC/校验（协议自带，错误必被拒）

| 层级 | 校验 | 保护对象 |
|------|------|---------|
| Slot Type | Golay(20,8,7) | 突发类型/色码 |
| LC Header | **Reed-Solomon(12,9,4)** 掩码 0x969696 | 72-bit LC（路径 A）|
| Embedded LC | **CS5 五位校验和** | 72-bit 重组 LC（路径 B）|
| EMB 头 | QR(16,7,6) | 16-bit EMB 头（CC/LCSS 对齐）|

这些是 ETSI 标准强制的纠错/校验码。若解调出的比特有错，校验在数学上几乎不可能通过
（CS5 误通过概率约 1/31，RS 更低），所以**"校验通过"本身就是正确性的强证据**。

### 两条独立路径交叉验证（cross-check）

- **路径 A（LC Header）**：数据同步突发 → BPTC(196,96) → RS(12,9,4)。
- **路径 B（Late Entry）**：语音超帧 → 4×EMB 碎片 → VBPTC(128,72) → CS5。

两条路径**用完全不同的突发、不同的交织/纠错算法、不同的校验码**，互不依赖。实测结果：

```
dmr_1: [A] SRC=1 DST=1 GroupVoiceChannelUser (Golay✓ RS✓)
       [B] SRC=1 DST=1 GroupVoiceChannelUser (CS5✓)   => 一致 ✔
dmr_2: [A] SRC=1 DST=1 GroupVoiceChannelUser (Golay✓ RS✓)
       [B] SRC=1 DST=1 GroupVoiceChannelUser (CS5✓)   => 一致 ✔
```

两条独立路径得到完全相同的 SRC/DST/FLCO，这种一致性极难由巧合产生，构成正确性的决定性证据。

### 跨超帧自洽

Late Entry 在 52–56 个独立超帧上反复解出**相同**的 SRC=1/DST=1（见 `late_entry_viz.png`）。
若解调有随机错误，结果会发散；稳定一致进一步佐证正确。

### 外部旁证

同一信号经 SDR++ → dsd-fme 能正常出语音，独立证明信号合法、协议解读无误。

### 关于 SRC=1/DST=1

这是该录音使用的真实 ID（测试/简单配置场景）。值虽小，但经上述四重验证确认是真实解码结果，
而非偶然或缺省值。
